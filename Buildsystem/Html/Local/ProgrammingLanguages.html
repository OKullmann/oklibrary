Programming languages used by the OKlibrary

Several formal languages are used in the library, for programming, mark-up and data description.

General programming

  1. C++ is the main general purpose language. We adhere strictly to the C++ standard (in the 2003-version ("C++03"), which for all practical purposes is the same as the 1998-version ("C++98")), except that we include the anticipated 2009-version ("C++09").

    Files containing C++ code either have the suffix .hpp (for header files) or the suffix .cpp (for implementation files).

  2. C is used only for the old OKsolver. We adhere strictly to the 1999-standard (C99).

    Files containing C code either have the suffix .h (for header files) or the suffix .c (for implementation files).

  3. The style of programming is modern C++, and not "just C with some add-ons" or the restricted point of view of object-orientation. However things should evolve, and especially at the start a good procedural C-style with extensions by concrete classes, exceptions and the standard library is recommended.
Programming languages like Perl or Python are not supported (they are considered as "false programming languages", in reality scripting languages for small jobs, but inflated to appear as "real programming languages"). For the purposes of prototyping we support languages from the realm of computer algebra (see below).

System programming

  1. Make is the system for the build system, and the associated programming language is accordingly the main programming language for the build system.

    Files containing make-programs are either called Makefile, or have the suffix .mak (while the name makefile is only used for deprecated makefiles).

  2. Bash or, more precisely, the associated programming language, is the programming language for system programming, used for meta-programs in the build-system, and for application test programs.

    Files containing Bash programs are not distinguished by a special naming convention, but by their starting line #!/bin/bash.

We consider Bash as a "true scripting language", which is used for relative short and easy programs. The use of Bash together with the Unix/Linux command line tools is unavoidable, and then the effort to learn for example Perl is wasted time (besides the point, that the character of Perl as the "private language of Larry" was finally to appalling for OK).

The Make-system appears old-fashioned, but its full power, based on its Lisp-like character, is not reached by any of the more modern system, while this power is essential for a holistic library like the OKlibrary.
