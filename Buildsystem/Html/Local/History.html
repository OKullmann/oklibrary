The history of the OKlibrary

General history

  • The initiation period: The "prehistory" is the development of the OKsolver:
    1. On 5.3.1998, the development started, based on the Boehm solver.
    2. The active development of the OKsolver finished August 2001. (Later only extensions to the user interface have been carried out.)
    3. The solver participated in the SAT 2002 competition and won two first ("random" category) and two third ("handmade" category) prizes.
    The original OKsolver, in refurbished form, is now part of the OKlibrary.
  • The experimentation period: From 2002 to 2004, exploration of C++ started, and many experiments were carried out with regard to generic programming.
  • The foundation period: February 2004 the EPSRC project "An algorithmic platform for efficient satisfiability-based problem solving" started, which created the foundations for the OKlibrary:
    1. the grant proposal (February 2003)
    2. the final report, official (short) version (August 2007; with a few editorial changes)
    3. the final report, unofficial, long version (September 2007).
  • The installment period: September 2007 version 0.2 of the OKlibrary was released. The basic foundation is laid, analogous to the planning of a new city, and until version 1.0 now (anticipated for the end of 2009) now the task is to implement (and extend) the plans, i.e., to "build the city".

Version history

  1. Version 0.2: 24.9.2007 (preparing the alpha-release planned for version 0.2.2, while the beta-release is planned for version 0.2.4, and the "official release" is planned for version 0.3).
  2. Version 0.2.1: 27.9.2008 (most of the basic targets for the alpha release are fulfilled, and now finishing and cleaning-up is needed).

Package history

  • The package-list shows all package meta-data, with lines for every package-release, starting with 
    running-number version SHA date
    where "SHA" is the git-hash-value of the repository-history, followed by pairs 
    md5sum package-name
    of md5sum-hash-values of the three possible packages (minimal, normal, and full).
  • For the changes introduced with each package, see the package history.
